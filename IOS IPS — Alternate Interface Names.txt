QUESTION 20 IOS IPS — Alternate Interface Names

Note: Interface names differ from Q11 — Gig0/1 instead of Gig0/0, se0/0/0 instead of se0/1/0. All IPS logic is identical.
 
-------------STEPS
Step 1 — Configure IPs and OSPF routing on all routers
R1(config)#
interface gig0/1
ip address 192.168.1.1 255.255.255.0
no shutdown
exit
interface se0/0/0
ip address 10.1.1.1 255.255.255.252
no shutdown
exit
router ospf 1
network 192.168.1.0 0.0.0.255 area 0
network 10.1.1.0 0.0.0.3 area 0
exit

R2(config)#
interface se0/0/0
ip address 10.1.1.2 255.255.255.252
clock rate 128000
no shutdown
exit
interface se0/0/1
ip address 10.2.2.2 255.255.255.252
clock rate 128000
no shutdown
exit
router ospf 1
network 10.1.1.0 0.0.0.3 area 0
network 10.2.2.0 0.0.0.3 area 0
exit

R3(config)#
interface gig0/1
ip address 192.168.3.1 255.255.255.0
no shutdown
exit
interface se0/0/0
ip address 10.2.2.1 255.255.255.252
no shutdown
exit
router ospf 1
network 192.168.3.0 0.0.0.255 area 0
network 10.2.2.0 0.0.0.3 area 0
exit


Step 2 — Enable security package on R1 and reload
R1(config)#
license boot module c1900 technology-package securityk9
! Type yes when prompted

R1#
copy running-config startup-config
reload
! After reload log back in and enter configure terminal


Step 3 — Set clock, create IPS directory, configure IPS rule
R1#
clock set 10:20:00 10 january 2014
mkdir ipsdir
! Press Enter to confirm
R1(config)#
ip ips config location flash:ipsdir
ip ips name iosips
ip ips notify log
service timestamps log datetime msec
logging host 192.168.1.50


Step 4 — Configure signature categories
R1(config)#
ip ips signature-category
category all
retired true
exit
category ios_ips basic
retired false
exit
exit
! Press Enter to confirm


Step 5 — Apply IPS rule to Gig0/1 outbound (note: gig0/1 not gig0/0)
R1(config)#
interface gig0/1
ip ips iosips out
exit


Step 6 — Modify signature 2004
R1(config)#
ip ips signature-definition
signature 2004 0
status
retired false
enabled true
exit
engine
event-action produce-alert
event-action deny-packet-inline
exit
exit
exit
! Press Enter to confirm
 

=====================================VERIFICATION
R1# show ip ips interfaces → GigabitEthernet0/1 Outbound: iosips
PC-C ping PC-A → Timeout (ICMP dropped by sig 2004)
PC-A ping PC-C → Success (reply not blocked)
Syslog (192.168.1.50) → SYSLOG tab → %IPS-4-SIGNATURE: Sig:2004 visible
 

KEY POINTS
• Only interface names change vs Q11. 'ip ips iosips out' goes on Gig0/1 (not Gig0/0).
• Serial interfaces are se0/0/0 and se0/0/1 in this question — adjust accordingly
